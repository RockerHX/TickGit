# 更新日志

## v1.7.1 - 2026-06-25

### 新增

- 新增页面文本选择偏好：开启后可拖拽选择仓库、分支、Commit、文件路径和 Diff 文本。
- 新增仓库右键菜单：支持复制仓库名称/路径、在 GitHub 查看、在 Terminal 打开、在文件管理器中显示、用 VS Code 打开和移除仓库。
- 新增分步推送 `preparing` 准备状态文案，用于顶部进度浮层。

### 修复

- 分支切换因本地未提交修改或未跟踪文件会被覆盖而失败时，展示结构化原因、目标分支和处理建议。
- 分步推送预览会先显示 loading 弹窗，再开始耗时的 Git 计划计算，减少“点击后没反应”的观感。
- 分步推送预览计划改为后台阻塞任务生成，避免同步 Git 计算拖慢窗口交互。
- 点击“开始分步推送”后快速返回 job id，将分支校验和 hash 二次校验移到后台，并立即显示准备中 overlay。
- 分步推送准备中也视为活跃任务，禁用可能冲突的切分支、推送、刷新和右键菜单操作。

### 调整

- 明确分步推送预览 loading 文案，提示正在检查远端状态并生成预览。
- 为分步推送预览生成增加阶段级性能日志；设置 `TICKGIT_PERF=1` 后可查看各关键阶段耗时。
