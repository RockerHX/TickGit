# GitHub Desktop 高保真 UI 重构任务

## 1. 目标

本任务只定义 TickGit 前端界面的视觉与布局重构，目标是让当前 UI 从“深色卡片化仪表盘”收敛为接近 GitHub Desktop 的“紧凑、扁平、三栏式桌面工作台”。

本任务只允许修改展示层：

- `src/routes/+page.svelte`
- `src/lib/components/*`
- `src/app.css`

以下内容不在本任务范围内：

- Git 逻辑
- Tauri command / event
- DTO / 类型结构
- 仓库持久化
- Push、分步提交、分页、选择、拖拽添加等业务行为

---

## 2. 参考图分析结论

### 2.1 GitHub Desktop 参考图的核心特征

参考图一的重点不是“深色主题”本身，而是以下工作台特征：

- 扁平化工具界面，而不是品牌化展示页
- 信息密度高，单屏承载更多可操作内容
- 顶部是紧凑工具栏，不是大面积展示型 Header
- 主体区域依靠明确分栏和 1px 分隔线组织内容
- 视觉重心来自层级、对齐和间距，而不是大圆角、强阴影、强渐变
- 选中态使用清晰但克制的蓝色高亮整行

### 2.2 当前 TickGit UI 与参考图的主要偏差

当前实现偏向品牌化卡片界面，和 GitHub Desktop 的工作台感差距明显：

- 背景渐变、发光和玻璃感过重
- 大圆角卡片过多，组件像多个独立模块而不是同一工作区
- 顶部区域体积过大，更像 Dashboard 而不是桌面工具栏
- Commit 历史是彩色卡片堆叠，而不是紧凑列表
- 文件列表和 Diff 面板边界过厚，留白过大，信息密度偏低
- 视觉层级依赖卡片和阴影，不依赖分栏、分隔线和行选中态

---

## 3. 视觉约束

### 3.1 全局风格

实现时统一遵循以下约束：

- 去掉大面积蓝色渐变、霓虹感、玻璃感
- 主背景改为 GitHub Desktop 风格的深灰 / 炭黑工作区
- 主交互色以蓝色选中态为主，辅以中性灰描边
- 圆角收敛为小到中等，不再使用大量胶囊型大圆角卡片
- 阴影降到最弱，只保留必要层级提示
- 优先使用分隔线、区域对齐和密度建立层级

### 3.2 页面结构

页面整体改造为：

- 顶部：紧凑工具栏
- 下方主体：三栏工作区
  - 左侧：Commit 历史
  - 中间：变更文件列表
  - 右侧：Diff
- Commit 详情头部置于中右工作区顶部，作为二级信息头，不再做成独立大卡片

### 3.3 信息密度

需要整体提高界面密度：

- 缩小区块留白
- 缩小标题字号层级
- 缩小按钮与输入控件体积
- 强化列表行式布局
- 用 1px 分隔线替代厚重卡片边框

### 3.4 对齐原则

- 对齐的是布局、层级、密度、色板和控件语义
- 不要求逐像素复制 GitHub Desktop
- 保留 TickGit 名称与现有功能入口
- 不伪造当前数据模型中不存在的信息

例如：

- 参考图中的 “Last fetched” 不新增
- 没有比较分支能力时，不实现真实 Compare 功能
- 没有增删行统计时，不补造统计数字

## 4. 实施任务

### 4.1 顶部区域重构

目标：把当前大号 Header 改成 GitHub Desktop 风格的紧凑工具栏。

实施要求：

- 仓库名称、仓库切换器、分支信息、Push 按钮放入同一横向工具条
- 采用分段式工具栏布局，而不是多个独立发光卡片
- Push 区域保留当前按钮逻辑、禁用逻辑和计数显示
- 分支块保留当前分支和 upstream 信息，但样式收敛为桌面工具控件
- 仓库切换器外观改为更接近桌面应用原生下拉控件
- 若保留 TickGit 标识，只作为弱化品牌文字，不占据大面积空间

明确限制：

- 不新增任何业务字段
- 不改变按钮点击行为
- 不改变仓库切换和分支状态数据来源

### 4.2 历史列表重构

目标：把当前彩色卡片式 Commit 历史改为 GitHub Desktop 风格的紧凑列表。

实施要求：

- 使用扁平列表行替代大卡片
- 顶部增加仅视觉用途的 “Changes / History” 页签壳时，只允许 `History` 为激活态，`Changes` 为禁用静态态
- 选中 Commit 使用接近参考图的蓝色高亮整行
- 已推送 / 未推送状态改为克制的行内标识，不再使用大面积琥珀色背景
- 保留作者、相对时间、短 hash 等现有信息，但按列表行重新排版
- 滚动、分页加载、右键菜单触发逻辑保持原样

明确限制：

- 不新增真正的 Changes 页面
- 不改变历史加载方式
- 不改变右键菜单可用条件

### 4.3 中右内容区重构

目标：把当前“详情卡片 + 文件卡片 + Diff 卡片”改成 GitHub Desktop 风格的连续工作区。

实施要求：

- 采用“详情头部 + 文件列表列 + Diff 列”的连续布局
- Commit 详情头部缩成紧凑信息条，保留摘要、作者、邮箱、时间、完整 hash
- 文件列表改为扁平行结构，不再使用大卡片
- 文件状态标签缩小并弱化，作为列表辅助标识
- Diff 区域改成更贴近代码审阅面板的贴边布局
- 减少 Diff 容器的大圆角和过大内边距
- 保留现有纯文本 Diff 展示方式，不扩展语法高亮或新交互

明确限制：

- 不新增文件统计
- 不新增分栏拖拽
- 不新增代码审阅功能
- 不改变文件选择与 Diff 加载逻辑

### 4.4 次级浮层视觉收敛

目标：统一右键菜单、Toast、拖拽浮层、分步提交浮层的工具化视觉。

实施要求：

- 去掉强发光、强玻璃感和过大的圆角
- 改为更贴近桌面工具的深灰浮层样式
- 保持现有交互时机、事件来源和文案语义
- 让浮层与主界面风格统一，但不喧宾夺主

明确限制：

- 不改变浮层出现条件
- 不改变 Toast 生命周期
- 不改变分步提交状态流转

### 4.5 全局样式 token 收敛

目标：统一全局背景、滚动条、描边、圆角和色板，避免局部重构后风格割裂。

实施要求：

- 在 `src/app.css` 中重设全局背景和基础色
- 滚动条改为低存在感的中性深灰样式
- 减弱全局阴影和高饱和色块
- 统一控件圆角层级
- 统一面板边框和分隔线风格

## 7. 验收标准

### 7.1 视觉验收

以下结果需要同时满足：

- 顶部区域从“大卡片头图”变为“紧凑工具栏”
- 主体区域呈现明显的 GitHub Desktop 式三栏工作区
- Commit 历史不再是彩色卡片堆叠
- 中间文件列表与右侧 Diff 形成连续工作区，而不是多个独立大卡片
- 全局背景、描边、圆角、阴影明显收敛到扁平桌面工具风格
- 选中态、边界和信息密度更接近 GitHub Desktop

### 7.2 功能冒烟验收

以下行为必须保持不变：

- 仓库切换正常
- Commit 选择正常
- 文件选择正常
- Diff 切换正常
- Push 按钮启用 / 禁用状态不变
- 右键菜单动作入口不变
- 分步提交浮层仍正常出现
- Toast 仍正常出现
- 拖拽浮层仍正常出现

---

## 8. 默认假设

- 已锁定目标为高保真对齐 GitHub Desktop，而不是保留当前品牌化卡片风格
- 对齐的是视觉结构和交互语义，不是逐像素复制
- 对参考图里没有数据支撑的元素不做伪造
- 默认继续使用现有窗口壳，不把任务扩大到 Tauri 原生标题栏或窗口行为改造
